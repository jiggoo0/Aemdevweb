# 🔍 AI SEO CONTEXT — AEMDevWeb

⚠️ Controls SEO strategy, content structure, and Google-facing logic  
⚠️ SEO must serve **business results**, not vanity traffic

---

## 0. SEO PHILOSOPHY (NON-NEGOTIABLE)

> **SEO ≠ เขียนบทความยาว**
>
> **SEO = ทำให้ Google เข้าใจ + คนอยากคลิก + คนอยากทัก**

ถ้าเนื้อหา:

- ยาวแต่ไม่ตอบคำถาม
- ติดอันดับแต่ไม่มี Conversion
- เขียนเพื่อ Bot อย่างเดียว

❌ ถือว่า **SEO ล้มเหลว**

---

## 1. SEO GOAL HIERARCHY

1️⃣ Index ถูก  
2️⃣ Rank ได้  
3️⃣ Click  
4️⃣ Trust  
5️⃣ Conversion

> ถ้า Rank แต่ไม่ทัก = Fail

---

## 2. KEYWORD STRATEGY (THAI-FIRST)

### Focus:

- Intent-based keyword
- Pain-based search
- Commercial + Informational hybrid

✅ ตัวอย่าง:

- “ทำเว็บ Next.js ดีไหม”
- “เว็บช้า แก้ยังไง”
- “จ้างทำเว็บ SME”
- “เว็บโหลดเร็ว ปิดการขาย”

❌ ห้าม:

- Keyword ยัด
- เขียนเพื่อปริมาณ
- แปลอังกฤษตรงตัว

---

## 3. CONTENT STRUCTURE FOR SEO

### Mandatory Layout

1. H1 = คำถามหลัก
2. Intro = Pain + Promise
3. H2 = Answer ตรง
4. H3 = ขยาย + ตัวอย่างจริง
5. CTA = Soft Sell

> ทุกบทความต้องตอบว่า  
> “อ่านแล้ว ผมควรทำอะไรต่อ?”

---

## 4. E-E-A-T RULES

### Experience

- อ้างจากการทำงานจริง
- มี Case / Code / Flow

### Expertise

- อธิบายได้ลึก แต่ไม่อวดศัพท์

### Authority

- ใช้โปรเจกต์จริง
- ใช้ชื่อ AEMDevWeb / Unlink-TH

### Trust

- ไม่การันตีเวอร์
- บอกข้อจำกัดตรงไปตรงมา

---

## 5. TECH SEO GUIDELINES

- Next.js App Router
- Server Components
- Metadata API
- Sitemap.ts / robots.ts
- MDX Content
- PageSpeed ≥ 95

SEO Content ต้อง:

- Render ฝั่ง Server
- Crawl ได้
- ไม่มี JS block เนื้อหา

---

## 6. INTERNAL LINKING STRATEGY

- เชื่อม Blog → Service
- เชื่อม Case Study → Conversion Page
- Anchor text = ธรรมชาติ

❌ ห้าม:

- “คลิกที่นี่”
- ลิงก์มั่ว
- ลิงก์เพื่อ SEO อย่างเดียว

---

## 7. SEO TONE RULE

SEO ของ AEMDevWeb ต้อง:

- คุยเหมือนคนจริง
- ไม่เป็นตำรา
- ไม่สอนแบบครู

> “ผมเล่าให้ฟังจากของที่ผมเจอจริง”
